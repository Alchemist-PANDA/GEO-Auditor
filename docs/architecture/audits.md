# AI Readiness Audit & AEO Content Score Specification

This document details the architecture of the AI Readiness Audit system and specifies the predictive machine learning model used to compute the **AEO Content Score**.

---

## 1. The AEO Content Score (Predictive ML Model)

The AEO Content Score is a **supervised machine learning model** (classification or regression) trained on millions of crawled pages. Instead of relying on a static checklist, the model predicts the probability that a given page will be selected by an engine as a citation source for a target prompt.

```
                  ┌──────────────────────┐
                  │ Semantic Similarity  │ ─┐
                  └──────────────────────┘  │
                  ┌──────────────────────┐  │
                  │ Schema Markup JSON   │ ─┼──> [Supervised ML Model] ──> Content Score (0-100)
                  └──────────────────────┘  │
                  ┌──────────────────────┐  │
                  │ Heading Structure    │ ─┤
                  └──────────────────────┘  │
                  ┌──────────────────────┐  │
                  │ Query Fan-out Cover  │ ─┘
                  └──────────────────────┘
```

### 1.1 Input Features (5 Feature Categories)

The model extracts features across five primary dimensions:

1. **Semantic Alignment**: Computes the cosine similarity between the page's content embedding and the target prompt's vector embedding.
2. **Structured Data Usage**: Evaluates the presence of `JSON-LD` schemas, giving higher weight to structured schemas (e.g., `FAQPage`, `HowTo`, `Organization`, `AggregateRating`) over generic `Article` structures.
3. **Content Structure**: Analyzes paragraph lengths, heading densities, title alignments, and the placement of direct-answer structures (e.g., answering questions directly in the first paragraph of a section).
4. **Query Fan-out Patterns**: Measures how well the page's vocabulary covers the sub-queries generated by engines during retrieval. This is a key proprietary signal, as it requires the platform's panel dataset to resolve the sub-queries.
5. **Recency / Freshness**: Evaluates HTTP headers (e.g., `Last-Modified`) and metadata dates on the page.

### 1.2 Model Training & Target Labels
* **Target Label ($Y$)**: Binary classification label indicating if a page was cited in an active engine run for a target prompt cluster ($Y=1$) or not ($Y=0$).
* **Training Corpus**: The aggregated database of crawled pages and citation maps. The model is retrained monthly to adapt to shifts in answer engine citation patterns.

## 2. Free Public AEO Report (Lead-Gen Audit)

The public lead-generation tool provides a simplified snapshot of AEO performance, delivered via HTML email to conversion prospects.

### 2.1 Report Sections & Ternary Ratings

The audit report uses a ternary rating system (**Good / Fair / Poor**) across six key sections:

| Section | What is Evaluated | Sample Dashboard Output |
|---|---|---|
| **AEO Snapshot** | Overview of the four core metrics. | Overall rating: **Fair** (AEO Content Score: 52/100). |
| **Visibility Score** | Brand mention rate across ChatGPT, Perplexity, and Google AI Overviews. | Ranks 14th in the category (10.2% visibility). Top competitor has 42%. |
| **Sentiment** | Ratio of positive to negative mentions in parsed answers. | **Good** (82% positive). Sentiment themes emphasize integration quality. |
| **Cited Websites** | Leaderboard of top domains and pages cited for target prompts. | majorbrands.com (30% share), wikipedia.org (15%), client.com (2%). |
| **Blog AEO Score** | Performance score of a submitted URL against the predictive model. | Score: 41/100 (**Poor**). Missing structured FAQ schemas. |
| **Suggestions** | Prioritized actions based on detected content gaps. | Create content targeting the "alternative tools" prompt cluster. |

## 3. Paid Enterprise Content Optimization Dashboard

The Enterprise dashboard replaces ternary ratings with granular metrics, detailed breakdowns, and action items:

* **Signal Breakdown**: Visualization of scores across all five input feature categories (Semantic, Schema, Structural, Fan-out, Recency).
* **Network Benchmarking**: Compares the page's performance against the distribution of comparable pages in the Profound Network.
* **LangGraph Integration**: Selecting "Optimize Content" automatically generates a brief and sends it to the agent workflow queue.

## 4. Architectural Summary Table

| Dimension | [CONFIRMED] Findings | [INFERRED] Systems Behavior | [ASSUMPTION] System Limits | [RECOMMENDATION] Optimization |
|---|---|---|---|---|
| **Scoring Engine** | The AEO Content Score is a predictive model, not a rule checklist. | The model predicts citation probability based on positive/negative cases. | The model cannot predict new engine citation updates in real time. | Retrain the model on a monthly cadence using the fresh citation corpus. |
| **Audit Delivery** | The free report is sent via HTML email, not as a PDF. | Prospects prefer direct email reports over logging into dashboards. | Large HTML email payloads trigger spam filters. | Host detailed audit charts on secure URLs and embed previews in emails. |
| **Signal MOAT** | Query fan-out alignment requires tracking search grounding queries. | Pages covering internal sub-queries achieve higher citation rates. | Sub-queries change dynamically based on user search context. | Cluster sub-query terms and map page content to these clusters. |
| **Feature Extraction** | Signals cover structure, structured schemas, and recency. | Extracting features requires crawling the prospect URL during audits. | Heavy crawler requests trigger IP bans or firewall blocks. | Run crawls using residential proxies and parse DOM elements locally. |
