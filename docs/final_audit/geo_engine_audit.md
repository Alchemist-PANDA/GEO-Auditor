# GEO Engine Audit

An evaluation of the Generative Engine Optimization (GEO) prompt parsing, visibility algorithms, sentiment models, and recommendation logic.

## Detailed Technical Review

### 1. Fragile Brand & Competitor Matchers
* **The Code**: The substring check is:
  ```python
  matches_name = brand.name.lower() in resp.raw_text.lower()
  matches_domain = brand.domain.lower() in resp.raw_text.lower()
  ```
* **The Flaw**: There are no word boundaries (`\b`) or token checks.
* **The Consequences**:
  - A brand named **"Rho"** will match words like **"through"**, **"rhombus"**, **"Rhode Island"**, or **"rhomboid"**.
  - A competitor named **"Brex"** will match **"brexit"**.
  - This results in highly inflated or inaccurate visibility and Share of Voice metrics.

### 2. Extremely Simple Sentiment Model
* **The Code**: Sentiment is calculated in [service.py](file:///e:/Profound-cloning/backend/app/modules/analysis/service.py#L47-L53) as:
  ```python
  pos_words = r'(best|great|recommend|fast|secure|love|excellent|superior|top|amazing)'
  neg_words = r'(bad|slow|error|expensive|flaw|avoid|terrible|worst|fail|issue)'
  pos = len(re.findall(pos_words, text, re.IGNORECASE))
  neg = len(re.findall(neg_words, text, re.IGNORECASE))
  return (pos - neg) / (pos + neg)
  ```
* **The Flaw**: This is a dictionary-based counter from the 1990s. It ignores negations, adverb qualifiers, and grammar structures.
* **The Consequences**: The sentiment scores generated by this model are highly inaccurate. A sentence like "This tool is not great and has failed to prevent errors" yields negative sentiment, but "This tool is not recommended, avoid the error, it's terrible" yields incorrect scores. It should utilize VADER, TextBlob, or a dedicated NLP sentiment model.

### 3. Subjective Visibility Scoring Heuristic
* **The Code**: The visibility score for a single response is calculated as:
  ```python
  resp_vis = base_mention * position_weight * sentiment_weight * has_citation * model_weight
  ```
  Where:
  - `position_weight` = `1.0 + (1.0 - relative_pos)` (Ranges from 1.0 to 2.0)
  - `sentiment_weight` = `1.0 + sentiment_score` (Ranges from 0.0 to 2.0)
  - `has_citation` = `1.2` or `1.0`
  - `model_weight` = `0.5` to `1.0` depending on the LLM model.
* **The Flaw**: The ceiling is defined as `4.8` (`1.0 * 2.0 * 2.0 * 1.2 * 1.0`).
* **The Consequences**: If the LLM output has negative sentiment (sentiment_score = -1.0), the `sentiment_weight` is `0.0`. This makes the entire response visibility score `0.0`, even if the brand is cited as the first item in the list! This is a severe logical error where negative mentions erase visibility.
